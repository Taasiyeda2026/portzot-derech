התיקון הקודם לא פתר את התקלה.

צריך לבצע בדיקת שורש ולא רק להחזיר שדות ידנית.

התקלה:
בשלב הכנת פוסטר החקר חסרים שוב שדות:
- שמות התלמידות
- כיתה
- בית הספר
- מה המשוב שקיבלנו ומה שיפרנו

בנוסף, בלחיצה על "חזרה" המערכת חוזרת למסך/גרסה קודמת, והשדות שנוספו נעלמים שוב.

המסקנה:
כנראה שיש בקוד יותר ממקום אחד שמגדיר או מרנדר את שלב הפוסטר:
- template ישן
- component ישן
- defaultState ישן
- render function נוסף
- navigation/back handler שטוען state קודם
- localStorage שמדרס את המבנה החדש
- או קובץ אחר שעדיין בשימוש בפועל

מה לבצע עכשיו:

1. לא להסתפק בהוספת השדות במקום אחד.
צריך לאתר את כל המקומות בקוד שבהם מופיע שלב הפוסטר / poster / research poster / final poster / build poster.

2. לחפש בקוד:
- poster
- research
- final
- feedback
- students
- school
- class
- back
- previous
- localStorage
- defaultState
- initialState
- renderPoster
- posterStep

3. לזהות מהו המסך האמיתי שנטען בפועל כאשר נכנסים לשלב הפוסטר ומהו המסך שנטען לאחר לחיצה על "חזרה".

4. למחוק או לנטרל שימוש בתבנית ישנה.
לא יכול להיות ששני מקורות שונים מייצרים את אותו שלב.

5. ליצור מקור אמת אחד בלבד עבור שדות הפוסטר:
posterData או שם דומה, שיכלול:
- studentNames
- className
- schoolName
- feedbackAndImprovements

6. לוודא שכל השדות האלה קיימים בכל המקומות:
- initialState / defaultState
- migration של localStorage
- render של שלב הפוסטר
- שמירה בזמן שינוי
- טעינה אחרי רענון
- מעבר קדימה
- חזרה אחורה
- תצוגה סופית / PDF אם קיים

7. לתקן את כפתור "חזרה":
כפתור חזרה צריך רק לשנות את currentStep לשלב הקודם.
הוא לא אמור:
- לאפס state
- לקרוא defaultState מחדש
- לטעון template ישן
- לדרוס localStorage
- לבצע reload
- להחזיר נתונים מגרסה קודמת

8. להוסיף console.log זמני לבדיקה:
- כאשר נכנסים לשלב הפוסטר
- כאשר לוחצים חזרה
- כאשר נטען state מ-localStorage
- כאשר נשמר state
כדי לוודא מאיפה נטענים הנתונים בפועל.

9. לבצע בדיקת רגרסיה:
- לפתוח את המערכת
- להגיע לשלב הפוסטר
- לוודא שכל ארבעת השדות מופיעים
- למלא נתונים
- לעבור קדימה
- ללחוץ חזרה
- לוודא שהשדות לא נעלמים
- לרענן את הדף
- לוודא שהשדות עדיין קיימים
- לבדוק שאין שום חזרה לגרסה קודמת

חשוב:
אם השדות חוזרים להיעלם אחרי "חזרה", סימן שהתיקון נעשה בקובץ הלא נכון או שיש render/template נוסף שעדיין פעיל.
יש לאתר את נקודת הרינדור האמיתית בפועל ולתקן אותה.
