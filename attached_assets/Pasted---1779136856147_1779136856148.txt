המשימה לא הושלמה במלואה.

קרא את הסיכום הקודם שלך והמשך מאותה נקודה בלבד.
אין לשכתב מחדש את התיקונים שכבר בוצעו, אלא אם הם שוברים בדיקות או גורמים לבאג ברור.

מה כבר בוצע ולא צריך לפתוח מחדש אלא רק לוודא שלא נשבר:
1. proposals-agreements — activity_names נשמר כ-text[] עם multi-select, labels בלבד ו-dedupe.
2. api.js — נרמול activity_names למערך מחרוזות.
3. permissions — הוסר הטקסט “ניהול גישה ודגלי הרשאה — תואם לגיליון permissions”.
4. Drawer — פורמט יום השבוע בתאריכי מפגשים עודכן ל“יום א׳”.
5. Service Worker — עודכנה גרסת cache / entry.

כעת יש להשלים את כל הסעיפים שלא בוצעו.

הנחיות מחייבות:
- אין לבצע שינוי SQL.
- אין לשנות את ההידר העליון.
- אין לשנות את סרגל הניווט הימני.
- אין לבצע zoom כללי לעמודים.
- אין לשנות routing או הרשאות קיימות.
- אין לשבור את activity_names שכבר תוקן.
- יש לבצע תיקונים ממוקדים בלבד.
- יש להשתמש בצבעי theme / CSS variables / בורר הצבע הקיים, ללא צבעים קשיחים.

יש להשלים את הסעיפים הבאים:

1. לוח הבקרה
- להעביר את תגית/תיבת “סיכום” למרכז מעל כרטיסי הסיכום.
- לשמור את כרטיסי הסיכום בפריסה של 2 שורות.
- להגדיל מעט את רוחב ה-container הפנימי בלי למתוח לרוחב מלא.
- לשפר את אזור “פעילויות לפי מחוזות”: פחות חללים ריקים, גובה מאוזן, תוכן קרוב יותר לכותרת, מחוזות ברורים יותר.
- להכהות טקסטים קטנים ואפורים מדי.
- לאחד שפת עיצוב של כרטיסים וקופסאות.
- לצמצם ריווח אנכי בין אזורים.
- לתקן את כרטיס “חריגות תפעוליות” כך שיספור רק את החריגות שמוגדרות בעמוד “חריגות”, באמצעות אותה לוגיקה בדיוק.
- להוסיף ניווט מסונן בלחיצה על כרטיסי הסיכום:
  - פעילויות → עמוד פעילויות.
  - חריגות תפעוליות → עמוד חריגות עם סינון החריגות האמיתי.
  - חסר מדריך → פעילויות ללא מדריך.
  - חסר תאריך התחלה → פעילויות ללא תאריך התחלה.
  - תאריכי סיום / מסתיימות החודש → עמוד תאריכי סיום עם החודש הנוכחי.
  - מדריכים → עמוד מדריכים.
  - סוגי פעילות כמו סדנאות / תוכניות / סיורים → עמוד רלוונטי מסונן.
- הכרטיסים צריכים להיראות לחיצים: cursor pointer ו-hover עדין ללא שינוי גודל.

2. עמוד הצעות — עיצוב
- להמשיך את תיקוני העיצוב שלא בוצעו.
- ליצור container פנימי קומפקטי וממורכז ברוחב max-width בערך 980px–1100px.
- כרטיס הכותרת, שורת הסינון והטבלה יהיו באותו רוחב פנימי.
- להקטין שדות בטופס ל־280px–320px.
- לסדר טופס בשתי עמודות סימטריות בדסקטופ.
- הערות יתפרסו על שתי העמודות בלבד.
- שם הפעילויות יכול להיות ברוחב מלא אם הוא multi-select.
- שורת הסינון תהיה קומפקטית: חיפוש 280px–360px, select 160px–220px.
- טבלת רשומות תהיה קומפקטית, ממורכזת, עם עמודות מבוקרות.
- להוסיף לטבלה מתאר מקצועי: border, border-radius 12px–16px, shadow עדין, כותרת “רשומות” כחלק מהמסגרת, רקע עדין לשורת כותרות וקווי הפרדה עדינים.
- גם במצב “אין רשומות להצגה” הטבלה תישאר מעוצבת.

3. עמוד תאריכי סיום פעילויות
- למרכז כותרת “תאריכי סיום פעילויות”.
- צבע הכותרת יגיע מה-theme / בורר הצבע.
- להסיר תגית “71 פעילויות”.
- להציג badge בפורמט “חודש מאי 30” לפי החודש הנוכחי בפועל.
- לספור רק פעילויות עם תאריך סיום תקין, בחודש הנוכחי ובשנה הנוכחית.
- לא לספור פעילויות ללא תאריך סיום או מחודשים אחרים.
- להגדיר container קומפקטי max-width בערך 1050px–1150px.
- לצמצם רוחבי עמודות:
  - שם פעילות 260px–320px
  - בית ספר 220px–260px
  - רשות 160px–200px
  - תאריך סיום 120px–150px
  - פעולה/סימון 50px–70px
- לעצב את הטבלה כ-card מקצועי עם border, radius, shadow, כותרות כהות וקווי הפרדה עדינים.
- להוסיף סינון select בשם “מנהל פעילויות”.
- ברירת מחדל: “כל מנהלי הפעילויות”.
- הרשימה תכלול רק מנהלים שקיימים בפועל, ללא כפילויות.
- בחירת מנהל תציג רק פעילויות שלו שמסתיימות בחודש הנוכחי.
- badge החודש והכמות יתעדכן בהתאם לסינון.

4. עמוד ארכיון פעילויות
- להסיר מהטבלה החיצונית את עמודת “תאריכי מפגשים”.
- להשאיר מידע זה רק בפרטי פעילות / Drawer / Popup אם נדרש.
- לצמצם את רוחב הטבלה ולעבור לעמודות מבוקרות.
- העמודות שנותרות:
  - תוכנית / סוג
  - רשות
  - בית ספר
  - מדריך
  - מנהל פעילות
  - תאריך התחלה
  - תאריך סיום
- כפתורי 2025 / 2026 / הכל יהיו ממורכזים יחד באותה שורה, עם רווח אחיד קטן.
- לעצב את מצב פעיל לפי בורר הצבע.
- להוסיף סינון בלחיצה על כרטיסי הסיכום העליונים.
- לחיצה על כרטיס מסננת את הטבלה לפי הערך.
- לחיצה חוזרת על אותו כרטיס מבטלת את הסינון.
- הסינון יעבוד יחד עם סינון שנה וחיפוש.
- כרטיס פעיל יקבל border / outline בצבע theme ללא שינוי גודל.

5. עמוד מדריכים
- למרכז כותרת “מדריכים פעילים”.
- צבע הכותרת מה-theme.
- badge כמות מדריכים ממורכז ונקי.
- להקטין רוחב כרטיסי מדריכים.
- כל הכרטיסים באותו רוחב, עם max-width ברור.
- למרכז את גריד הכרטיסים.
- שורה חלקית תישאר ממורכזת.
- להוסיף gap של 16px–24px.
- כאשר ערך מספרי הוא 0, להחליש רק את המספר עם opacity נמוך או אפור בהיר.
- מספרים מעל 0 יישארו ברורים.
- להחליף “כל התקופה” בניווט תקופה עם חצים בדומה ללוח הבקרה.
- ברירת מחדל: החודש הנוכחי.
- מעבר חודש יעדכן את נתוני הכרטיסים.
- סינון מנהל פעילות וחיפוש יעבדו יחד עם התקופה.
- לתקן חיפוש עם debounce של 250ms–350ms.
- בזמן הקלדה לא לאבד focus ולא לאפס סמן.
- לא לבצע רינדור מלא בכל keypress.

6. my-data למדריך
- להשאיר my-data כמסך ברירת מחדל של instructor.
- להפוך אותו למסך עבודה דומה לעמוד הפעילויות.
- להציג טבלה ברורה.
- להשתמש בלוגיקת api.myData הקיימת.
- להציג רק פעילויות שבהן המדריך המחובר מופיע ב-emp_id או emp_id_2.
- לא להציג למדריך:
  - מחיר
  - מימון
  - מידע כספי
  - הערות פרטיות
  - מידע תפעולי פנימי
- כאשר מדריך פותח פעילות ויש מדריך נוסף:
  - להציג רק את שם המדריך הנוסף.
  - לא להציג רשימת מדריכים כללית.
  - לא להציג פרטי קשר של מדריכים.
- אם המחובר הוא emp_id / instructor_name, להציג instructor_name_2 אם קיים.
- אם המחובר הוא emp_id_2 / instructor_name_2, להציג instructor_name אם קיים.
- אם אין מדריך נוסף, לא להציג שדה מיותר או להציג “אין מדריך נוסף” לפי שפת המערכת.
- לוודא ש-week ו-month למדריך מציגים רק את הפעילויות שלו.
- לוודא שאין cache שמציג למדריך נתונים של משתמש אחר.

7. Drawer / פרטי פעילות
- בנוסף לתיקון יום השבוע שכבר בוצע, להשלים:
- באזור “מפגשים ותאריכים” להציג באותה שורה:
  [4 מתוך 8 מפגשים · 50%] | [🏁 תאריך סיום 15/06/2026]
- בדסקטופ זה חייב להיות בשורה אחת.
- פס ההתקדמות, אם נשאר, יהיה עדין מתחת לשורה.
- להסיר את הרקע הכהה מהידר ה-Drawer.
- ההידר יהיה לבן / שקוף.
- להוסיף קו הפרדה תחתון עדין בצבע theme.
- שם הפעילות יהיה כותרת ראשית גדולה וברורה.
- שם התוכנית / סוג הפעילות יהיה טקסט משני קטן יותר.
- כפתורי פעולה עליונים יהיו מיושרים באותה שורה.
- להקטין את התיבות / תגיות בהידר:
  - רוחב לפי תוכן
  - padding סביר
  - לא width מלא
  - badge קטן ונקי
  - גובה קומפקטי ואחיד

8. רכיב ניווט שבוע / חודש
- אם עדיין לא בוצע:
- הניווט והכפתורים בצד יהיו באותה שורה.
- להקטין את כפתורי “שבוע” ו“חודש” בגובה וברוחב.
- הכפתורים יהיו קומפקטיים.
- לא להחזיר אותם להידר העליון.

9. Service Worker
- כבר עודכנה גרסה, אבל יש לוודא שהעדכון מלא:
  - install שומר רק קבצים נדרשים.
  - activate מוחק caches ישנים.
  - self.skipWaiting() קיים.
  - clients.claim() קיים.
  - HTML / JS / CSS לא נתקעים ב-cache ישן.
  - manifest ו-PWA תקינים.
- אין לשנות UI במסגרת סעיף זה.

  - שם הבדיקה.
  - למה היא אינה קשורה לתיקון.
- אם כשלון קשור לתיקון הנוכחי, לתקן אותו.
- לא לסיים עם כשלון בדיקה שקשור לשינויים שבוצעו.

בסיום להחזיר:
1. קבצים ששונו.

4. אילו בדיקות נכשלו ואם הן קשורות או לא קשורות לתיקון.
