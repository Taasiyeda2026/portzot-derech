צריך לתקן את עמוד עריכת הפוסטר והייצוא ל־PDF.

מה רואים כרגע:
1. הפוסטר מוצג קטן מדי בתוך המסך.
2. התוכן בתוך הפוסטר דחוס ולא קריא.
3. נראה שה־scale / zoom של התצוגה משפיע על הפוסטר עצמו ולא רק על ה־preview.
4. אם ה־PDF מיוצא מתוך אותו מצב, הוא יוצא באיכות/פריסה לא תקינה.
5. בנוסף יש לבצע עדכון Service Worker כדי לוודא שלא נטענים קבצים ישנים מה־cache.

דרישות תיקון:

1. להפריד לגמרי בין גודל אמיתי של הפוסטר לבין גודל התצוגה במסך:
   - הפוסטר עצמו צריך להישאר בגודל A4 אמיתי.
   - אין להפעיל transform, scale או zoom על האלמנט הפנימי של הפוסטר.
   - כל הקטנה למסך צריכה להיות רק על wrapper חיצוני של preview.

2. האלמנט שמייצג את הפוסטר לייצוא צריך להיות נקי:
   - width: 210mm
   - height: 297mm
   - margin: 0
   - padding לפי העיצוב
   - transform: none
   - zoom: 1
   - overflow: hidden רק אם נדרש לשמירה על גבולות A4.

3. בתצוגת העריכה במסך:
   - ליצור wrapper חיצוני שממרכז את הפוסטר.
   - עליו בלבד להפעיל scale כדי שהפוסטר ייכנס למסך.
   - לא לשנות את מידות הפוסטר עצמו.
   - אם צריך, להשתמש ב־transform-origin: top center.
   - לוודא שה־wrapper מקבל גובה מחושב בהתאם לסקייל כדי שלא יהיו קפיצות גלילה.

4. בייצוא PDF דרך Puppeteer:
   - לשלוח לשרת HTML נקי של הפוסטר ללא מצב עריכה, ללא כפתורים וללא wrapper של preview.
   - לוודא שב־HTML שנשלח אין scale / zoom / transform.
   - להוסיף CSS ייעודי לייצוא:
     @page { size: A4 portrait; margin: 0; }
     html, body { margin: 0; padding: 0; width: 210mm; min-height: 297mm; }
   - להפעיל emulateMediaType('print').
   - לוודא שה־PDF מיוצא כעמוד A4 אחד ולא כתמונה קטנה בתוך עמוד.

5. איכות:
   - לוודא שתמונות בפוסטר לא נמתחות בצורה לא פרופורציונלית.
   - להשתמש ב־object-fit: cover או contain לפי אזור התמונה.
   - לא לייצא צילום מסך מטושטש אלא רינדור HTML אמיתי דרך Puppeteer.

6. עדכון Service Worker:
   - להעלות גרסת cache חדשה, לדוגמה CACHE_NAME = 'poster-builder-vX'
   - באירוע activate למחוק caches ישנים.
   - לוודא ש־admin.html, editor.html, קבצי JS ו־CSS לא נשארים בגרסה ישנה.
   - אם יש בעיית cache בפיתוח, להוסיף אפשרות bypass או network-first לקבצים קריטיים.
   - לאחר העדכון, לוודא שהדפדפן טוען את הקבצים החדשים ולא את הישנים.

7. בדיקה לאחר תיקון:
   - לפתוח את עמוד editor.html?type=physical.
   - לוודא שהפוסטר נראה ברור, מאוזן, לא דחוס ולא קטן מדי.
   - ללחוץ ייצוא PDF.
   - לוודא שה־PDF יוצא בעמוד A4 אחד, חד, לא לבן, לא חתוך ולא מוקטן.
   - לוודא שאין שגיאות בקונסול ואין שגיאות בשרת.
   - לוודא שעדכון Service Worker נכנס לפועל אחרי refresh קשה או פתיחה מחדש.
