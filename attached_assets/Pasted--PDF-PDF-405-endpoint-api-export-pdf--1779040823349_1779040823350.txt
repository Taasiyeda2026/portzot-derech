יש שגיאה בייצוא PDF:

ייצוא PDF נכשל (שגיאה 405).
ה-endpoint /api/export-pdf אינו זמין בכתובת זו.
יש לוודא שהאתר רץ דרך שרת Node.js ולא אחסון סטטי בלבד.

מה צריך לבדוק ולתקן:

1. לוודא שהאתר בפרודקשן לא מוגש כאחסון סטטי בלבד.
   כרגע נראה שהכתובת:
   https://portzot-derech.org/poster-builder/product/editor.html?type=physical
   לא מפעילה בפועל את server.js של Node עבור /api/export-pdf.

2. לבדוק שהקובץ server.js באמת רץ בפרודקשן.
   צריך לוודא שב־package.json יש:
   "start": "node server.js"
   או פקודת start מתאימה שמריצה את שרת ה־Node ולא רק שרת סטטי.

3. לוודא שקיים route פעיל:
   POST /api/export-pdf

   לדוגמה:
   app.post('/api/export-pdf', async (req, res) => {
     ...
   });

4. לוודא ש־express.json מוגדר לפני ה־route:
   app.use(express.json({ limit: '50mb' }));

5. להוסיף endpoint בדיקה פשוט:
   GET /api/health

   שיחזיר:
   { ok: true }

   כדי שנוכל לבדוק מהדפדפן אם שרת ה־Node באמת פעיל בכתובת הציבורית.

6. אם האתר מוגש משרת סטטי וה־Node רץ במקום אחר:
   - ליצור כתובת API נפרדת, למשל:
     https://api.portzot-derech.org/api/export-pdf
   - או להעביר את כל האתר לרוץ דרך אותו שרת Node.
   - לעדכן את הקריאה בצד לקוח כך שתשלח לכתובת הנכונה.

7. אם משתמשים ב־Replit:
   - לוודא שה־deployment מריץ את server.js.
   - לוודא שהדומיין portzot-derech.org מחובר ל־deployment של Node ולא לעותק סטטי.
   - לבדוק שה־Run / Start command הוא Node server ולא פתיחת HTML סטטי.

8. לעדכן את קריאת ה־fetch בצד לקוח:
   - אם השרת והאתר באותה כתובת:
     fetch('/api/export-pdf', { method: 'POST', ... })
   - אם ה־API בכתובת אחרת:
     fetch(API_BASE_URL + '/api/export-pdf', { method: 'POST', ... })

9. לוודא שה־Service Worker לא מתערב בבקשות API:
   - כל בקשה שמתחילה ב־/api/ צריכה לעבור ישירות ל־network.
   - לא לשמור /api/export-pdf ב־cache.
   - לא להחזיר עבורו קובץ HTML סטטי.

10. להוסיף הודעת שגיאה ברורה בצד לקוח:
   אם מתקבלת שגיאה 405 או 404:
   "שרת ייצוא ה־PDF אינו פעיל בכתובת הנוכחית. יש להריץ את האתר דרך שרת Node.js או לעדכן את כתובת ה־API."

בדיקות לאחר תיקון:
1. לפתוח בדפדפן:
   /api/health
   ולוודא שמתקבל JSON עם ok:true.
2. לפתוח את עמוד הפוסטר.
3. ללחוץ ייצוא PDF.
4. לוודא שנשלחת בקשת POST ל־/api/export-pdf.
5. לוודא שהתגובה היא application/pdf.
6. לוודא שה־PDF יורד תקין, לא לבן, לא חתוך ולא מוקטן.
